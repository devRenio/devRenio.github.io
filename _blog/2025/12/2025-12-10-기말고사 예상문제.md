## 객체지향프로그래밍II(김정준) 기말고사 예상문제

### 1. 가상 함수와 오버라이딩

1.  **가상 함수(Virtual Function)의 정의와 선언 방법은 무엇인가?**<br>(virtual 키워드와 선언 위치 설명)
2.  **가상 함수 호출 시 발생하는 동적 바인딩(Dynamic Binding)이란 무엇인가?**<br>(컴파일 타임의 정적 바인딩과 대조하여 실행 시간에 일어나는 일 설명)
3.  **오버라이딩(Overriding)의 개념과 성립 조건은 무엇인가?**<br>(함수 재정의와의 차이점, 리턴 타입/매개변수 일치 조건 설명)
4.  **가상 소멸자(Virtual Destructor)는 왜 필수적으로 사용해야 하는가?**<br>(기본 클래스 포인터로 객체 삭제 시 메모리 누수 방지 설명)
5.  **결과적으로 가상 함수와 오버라이딩을 통해 어떻게 다형성(Polymorphism)을 실현하는가?**<br>(하나의 포인터로 여러 객체를 다루는 효과 요약)

### 2. 추상 클래스 (순수 가상 함수)

1.  **순수 가상 함수(Pure Virtual Function)란 무엇이며 어떻게 선언하는가?**<br>(함수의 몸체가 없고 =0으로 선언됨을 설명)
2.  **추상 클래스(Abstract Class)의 정의와 인스턴스화 제약은 무엇인가?**<br>(최소 하나의 순수 가상 함수 포함, 객체 생성 불가 설명)
3.  **추상 클래스를 상속받은 파생 클래스는 어떤 의무를 가지는가?**<br>(순수 가상 함수를 반드시 오버라이딩해야 함을 설명)
4.  **추상 클래스는 상속 관계에서 어떤 목적(역할)으로 사용되는가?**<br>(기본 클래스 역할, 통일된 인터페이스 제공 설명)
5.  **추상 클래스를 이용한 구현의 예시는 어떠한가?**<br>(Shape 클래스와 이를 상속받은 Circle, Rect의 예시 코드 작성)

### 3. 템플릿 (함수 템플릿 및 클래스 템플릿)

1.  **일반화(Generalization)와 제네릭 프로그래밍이란 무엇인가?**<br>(데이터 타입에 의존하지 않는 로직 작성의 의미 설명)
2.  **함수 템플릿의 선언 방법과 작동 원리는 무엇인가?**<br>(template <typename T> 문법과 인자 추론 설명)
3.  **클래스 템플릿의 선언 방법과 객체 생성 방식은 무엇인가?**<br>(제네릭 멤버 변수와 명시적 타입 지정 <int> 등 설명)
4.  **컴파일러에 의한 템플릿의 구체화(Specialization) 과정은 어떻게 이루어지는가?**<br>(실제 호출 시점에 구체적인 코드가 생성됨을 설명)
5.  **템플릿을 사용했을 때의 장점은 무엇인가?**<br>(코드 재사용성 증가, 중복 제거, 형 안전성 등 요약)

### 4. 스트림과 버퍼

1.  **C++에서 스트림(Stream)이란 어떤 개념인가?**<br>(데이터의 흐름, 장치와 프로그램의 연결 통로 설명)
2.  **표준 입출력 스트림 객체(cin, cout 등)는 각각 무엇과 연결되는가?**<br>(키보드, 스크린 등 하드웨어와의 매핑 설명)
3.  **스트림 버퍼(Buffer)란 무엇이며 입출력 시 왜 사용하는가?**<br>(임시 저장소 개념, 저수준 시스템 콜 최소화를 통한 효율성 설명)
4.  **버퍼가 비워지는(Flushing) 구체적인 시점들은 언제인가?**<br>(버퍼 만료, \n, flush, 프로그램 종료 등)
5.  **스트림 기반 입출력 방식의 특징과 장점은 무엇인가?**<br>(장치 독립성, 문자 단위 처리 등 요약)

### 5. 포맷 입출력

1.  **포맷 입출력이란 무엇이며 어떤 경우에 필요한가?**<br>(데이터의 가독성 향상, 특정 양식 준수 필요성 도입)
2.  **포맷 플래그(Format Flags)를 이용해 형식을 지정하는 방법은 무엇인가?**<br>(setf, unsetf 함수와 ios::hex 등의 비트 플래그 설명)
3.  **포맷 함수(Format Functions)를 이용해 너비나 정밀도를 조절하는 방법은 무엇인가?**<br>(width, precision, fill 멤버 함수 설명)
4.  **조작자(Manipulator)란 무엇이며 다른 방식과 비교했을 때의 장점은 무엇인가?**<br>(<< 연산자와 함께 쓰는 편리함, endl 등 설명)
5.  **매개변수가 있는 조작자(iomanip)는 어떻게 사용하는가?**<br>(setw, setfill 등의 사용법과 헤더 파일 포함 설명)

### 6. C++ 예외 처리

1.  **예외(Exception)란 무엇이며 기존의 오류 처리(if문 등)와 어떻게 다른가?**<br>(실행 중 발생하는 예기치 못한 상황, 탐지와 처리 코드의 분리 설명)
2.  **try-throw-catch로 이어지는 예외 처리의 기본 구조와 흐름은 어떠한가?**<br>(감시 -> 던지기 -> 받아서 처리하기의 단계 설명)
3.  **기본 타입 외에 객체(예외 클래스)를 예외로 던지고 받는 방법은 무엇인가?**<br>(사용자 정의 예외 클래스 활용 및 정보 전달 설명)
4.  **예외 발생 시 스택 풀기(Stack Unwinding)란 무엇인가?**<br>(적절한 catch 블록을 찾을 때까지 함수가 종료되며 스택이 정리되는 과정 설명)
5.  **예외 처리를 도입함으로써 얻을 수 있는 이점은 무엇인가?**<br>(프로그램의 비정상 종료 방지, 유지보수성 향상 요약)

### 7. 람다식 (Lambda Expression)

1.  **람다식(Lambda Expression)의 정의와 주된 사용 목적은 무엇인가?**<br>(이름 없는 함수, 일회성 코드 작성의 간결함 설명)
2.  **람다식의 4가지 핵심 구성 요소(캡쳐, 매개변수, 리턴, 바디)는 어떻게 작성하는가?**<br>([], (), ->, {} 문법 구조 설명)
3.  **캡쳐 리스트(Capture List)를 통해 외부 변수를 사용하는 방법(값 vs 참조)은 무엇인가?**<br>([=], [&], [var] 등의 차이 설명)
4.  **auto 키워드는 람다식과 어떻게 함께 사용되는가?**<br>(람다 타입의 추론 및 변수 저장 방법 설명)
5.  **STL 알고리즘(for_each 등)에서 람다식은 어떻게 활용되는가?**<br>(함수 포인터나 함수 객체 대신 코드를 인라인으로 넘기는 예시 작성)
